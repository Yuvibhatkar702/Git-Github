Command In Git

  • git init :- "git init" is a Git command that creates a new, empty Git repository in the current directory, initializing the necessary files and directories to begin tracking changes to files.
  • git clone "https-link" :- The "git clone" command is used to create a local copy of a remote Git repository, which allows you to work on the project files on your own machine.
  • git add . :- "git add" is used to stage changes made to files, preparing them to be committed to the Git repository. It differs from other Git commands in that it does not actually commit the changes, but rather adds them to the staging area, ready for the next commit.
  • git commit :- The `git commit` command is used to record changes made to the files in a Git repository.
  • git push :- "git push" is a Git command used to upload local repository content to a remote repository. It allows you to send the committed changes in your local branch to the remote repository.
  • git branch :- is a command that creates a new, independent line of development within a Git repository, allowing developers to work on different features or fixes simultaneously without affecting the main codebase.
  • git log :- Git log is a command that allows you to view the commit history of a Git repository, including the commit messages, authors, and timestamps, providing a detailed record of changes over time.
